# LangGraph의 핵심: 상태(State) 관리

LangGraph는 여러 AI 에이전트나 도구들을 연결하여 복잡한 작업을 수행하는 라이브러리입니다. 일반적인 LangChain 체인과 가장 큰 차이점은 '상태(State)'를 중심으로 모든 작업이 이루어진다는 점입니다.

## StateGraph란?

StateGraph는 LangGraph에서 워크플로우를 정의하는 핵심 클래스입니다. 이름 그대로, '상태(State)'를 가진 '그래프(Graph)'를 의미합니다.

- **상태 (State)**: 그래프의 전체 작업 흐름 동안 유지되는 데이터 저장소입니다. 보통 Python의 TypedDict를 사용하여 어떤 데이터가 저장될지 명확하게 정의합니다. 예를 들어, 사용자의 질문, 중간 결과, 최종 답변 등을 상태에 저장할 수 있습니다.
- **그래프 (Graph)**: 작업의 흐름을 나타냅니다. 각 작업 단계를 '노드(Node)'라고 부르며, 노드와 노드 사이를 '엣지(Edge)'로 연결하여 작업 순서를 결정합니다.

StateGraph를 사용하면, 각 노드가 이전 노드에서 변경한 '상태'를 그대로 넘겨받아 다음 작업을 이어나갈 수 있습니다. 이를 통해 여러 단계에 걸친 복잡한 대화나 작업 이력을 관리하는 것이 매우 용이해집니다.
